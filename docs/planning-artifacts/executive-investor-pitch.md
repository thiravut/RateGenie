## 1. บทสรุปผู้บริหาร (Executive Summary)

**RateGenie** คือ AI Revenue Assistant ที่ทำให้โรงแรมทุกขนาดในประเทศไทยเข้าถึงความเชี่ยวชาญด้าน Revenue Management ระดับพรีเมียมได้ ผ่าน AI ที่ฝึกจากผู้เชี่ยวชาญจริง ส่งคำแนะนำผ่าน LINE — แอปที่คนไทยใช้ทุกวัน

### ปัญหาที่แก้

- **48%** ของโรงแรมไทยสูญเสียรายได้อย่างน้อยสัปดาห์ละครั้ง เพราะปรับราคาไม่ทันสถานการณ์
- **90%+** ของโรงแรมอิสระไม่มีระบบ Revenue Management System (RMS)
- ระบบ RMS นานาชาติราคาแพง (€119–$2,000+/เดือน) ภาษาอังกฤษ ไม่เข้าใจบริบทตลาดไทย

### โซลูชัน

RateGenie เป็น **AI ที่เข้าใจตลาดไทย** — เทศกาลสงกรานต์ ลอยกระทง ช่วง High/Low Season — แนะนำราคาพร้อม **เหตุผลเป็นภาษาไทย** ส่งตรงถึงเจ้าของโรงแรมผ่าน **LINE** ด้วยหลักการ **"AI แนะนำ คุณตัดสินใจ"**

### ตำแหน่งการแข่งขัน

**ไม่มีคู่แข่งรายใดในตำแหน่งนี้:** AI ตั้งราคา + ภาษาไทย + LINE-first + ราคาที่ SME จ่ายได้

## 3. โซลูชันของเรา (Our Solution)

### หลักการ: "AI แนะนำ คุณตัดสินใจ"

RateGenie ไม่ใช่ระบบอัตโนมัติที่ตั้งราคาเอง แต่เป็น **ผู้ช่วยอัจฉริยะ** ที่วิเคราะห์ข้อมูลและแนะนำพร้อมเหตุผล — เจ้าของโรงแรมเป็นคนอนุมัติ สร้างความไว้วางใจแบบค่อยเป็นค่อยไป (Progressive Trust Model)

### ประสบการณ์ใช้งานหลัก

```
┌─────────────────────────────────────────────────────┐
│              Morning Approval Journey               │
│                   (30 วินาที)                        │
│                                                     │
│   📱 LINE แจ้งเตือน 7:00 น.                         │
│       ↓                                             │
│   🔗 คลิกลิงก์ → เข้าสู่ระบบอัตโนมัติ                │
│       ↓                                             │
│   📊 ดูคำแนะนำ AI + เหตุผลเป็นภาษาไทย               │
│       ↓                                             │
│   ✅ แตะ [อนุมัติ] → "อนุมัติแล้ว!"                   │
```

## 4. รูปแบบธุรกิจ (Business Model)

### โมเดลรายได้: SaaS Subscription

| แพ็กเกจ | ราคา/เดือน | กลุ่มเป้าหมาย |
|---|---|---|
| **Starter** | ฿999 (~$27) | โรงแรมเดี่ยว 10–50 ห้อง |
| **Professional** | ฿1,499 (~$41) | 3–5 โรงแรม / Revenue Manager |
| **Enterprise** | ฿1,999+ (~$55+) | เครือโรงแรม ไม่จำกัดจำนวน |

> **เปรียบเทียบ:** RoomPriceGenie €119+/เดือน (~฿4,500+), IDeaS $500–2,000+/เดือน (~฿18,000–73,000+)
> **RateGenie ถูกกว่า 3–70 เท่า** พร้อมความได้เปรียบด้านภาษาไทยและ LINE

### Unit Economics

| ตัวชี้วัด | ค่า |
|---|---|
| **ARPU** (Average Revenue Per User) | ~฿1,500/เดือน ($41) |
| **ต้นทุนคงที่** (Server + Channex + AI API) | ~$146–177/เดือน |
| **Gross Margin เป้าหมาย** | 70%+ |
| **CAC** (Customer Acquisition Cost) | ต่ำ — ผ่าน word-of-mouth, Revenue Manager referral |
| **Revenue Manager Multiplier** | 1 RM → 5–15 โรงแรม (viral loop) |
| **Target LTV** | >24 เดือน (churn <5%) |

### ช่องทางการขาย

1. **Revenue Manager Referral** — Revenue Manager 1 คน → แนะนำ 5–15 โรงแรม (viral loop)
2. **Word-of-mouth** — เจ้าของโรงแรมบอกต่อ (ช่องทางหลักในตลาดไทย +14% awareness)
3. **Facebook Groups / LINE Groups** — ชุมชนเจ้าของโรงแรม
4. **Direct Sales** — โทรศัพท์ + demo ผ่าน LINE Video
5. **30 วันทดลองฟรี** — ดึงข้อมูล OTA อัตโนมัติ เห็นผลก่อนจ่าย

| 2 | **Thailand-First** | เข้าใจสงกรานต์ ลอยกระทง Long Weekend ปิดเทอม ตรุษจีน — คู่แข่งต่างชาติไม่มี |
| 3 | **LINE-Native** | Unfair advantage — ไม่มีคู่แข่งนานาชาติรายใดส่งคำแนะนำผ่าน LINE |
| 4 | **Progressive Trust** | "AI แนะนำ คุณตัดสินใจ" — ไม่บังคับ automation ลดความกังวลเรื่องการควบคุม |
| 5 | **ราคาที่ SME จ่ายได้** | ฿999–1,999/เดือน เทียบกับ €119–$2,000+ ของคู่แข่ง (ถูกกว่า 3–70 เท่า) |
| 6 | **Proactive Alerts** | ไม่รอให้เจ้าของเข้ามาดู แต่ push โอกาสไปถึงมือ |
| 7 | **Revenue Manager Multiplier** | RM 1 คน จาก 5 โรงแรม → 15 โรงแรม = viral growth loop |

### Moat (คูเมืองป้องกัน)

1. **ข้อมูลราคาสะสม** — ยิ่งใช้นาน ยิ่งมีข้อมูลตลาดไทยมาก AI ยิ่งแม่นยำ
2. **Feedback Loop** — ทุกครั้งที่เจ้าของอนุมัติ/ปฏิเสธ AI เรียนรู้ → ยากที่คู่แข่งจะ replicate
3. **Network Effect** — RM แนะนำโรงแรม → โรงแรมแนะนำ RM → วงจรที่ขยายตัวเอง
4. **Thai Market Context** — ความเข้าใจพฤติกรรมการจองของคนไทย เทศกาล ฤดูกาล ที่สะสมมาไม่ได้ข้ามคืน

## 10. ความเสี่ยงและแผนรับมือ (Risks & Mitigation)

| ความเสี่ยง | ระดับ | แผนรับมือ |
|---|---|---|
| OTA เปลี่ยน API กะทันหัน | ปานกลาง | ใช้ Channex เป็น abstraction layer — Channex จัดการ API changes |
| เจ้าของไม่ไว้ใจ AI ตั้งราคา | สูง | Progressive trust model: แนะนำ → อนุมัติ → กฎอัตโนมัติ + เหตุผลโปร่งใส |
| ไม่เห็น ROI เร็วพอ → Churn | สูง | Time-to-Value <14 วัน + Dashboard แสดงรายได้ก่อน/หลังชัดเจน |
| คู่แข่งนานาชาติเข้าตลาดไทย | ต่ำ | ยากที่จะ localize ได้ระดับเดียวกัน (ภาษาไทย + LINE + บริบทท้องถิ่น) |
| ปัญหา Data Privacy (PDPA) | ปานกลาง | ออกแบบ PDPA-compliant ตั้งแต่แรก: Tenant isolation, AES-256, Audit log |
| Scaling ไม่ทัน | ต่ำ | Modular Monolith → แยก service ตามคอขวดเมื่อถึงเวลา |
